# Responsible Use Policy

## Purpose

This project and its associated materials are developed and maintained strictly for the following purposes:

- **Education** -- learning about cybersecurity concepts, web application architecture, and secure development practices
- **Authorized security testing** -- conducting penetration tests, vulnerability assessments, and security audits with explicit written permission from the system owner
- **Capture The Flag (CTF) competitions** -- participating in sanctioned cybersecurity competitions and training exercises
- **Academic and independent research** -- advancing the understanding of security topics in a controlled, lawful manner

## Prohibited Uses

The following uses of this project are strictly prohibited:

- **Unauthorized access** -- using any tools, techniques, or knowledge derived from this project to access systems, networks, or data without explicit authorization from the owner
- **Malware development or distribution** -- adapting, incorporating, or leveraging any part of this project to create, distribute, or deploy malicious software
- **Data theft or exfiltration** -- using this project to steal, intercept, or exfiltrate personal, corporate, or governmental data
- **Denial of service** -- employing this project to disrupt the availability of any system, service, or network
- **Fraud or deception** -- using this project to impersonate individuals, organizations, or services for fraudulent purposes
- **Violation of any applicable law** -- any use that contravenes local, national, or international laws and regulations

Users must ensure that their activities comply with all relevant legal frameworks, including but not limited to:

- Computer Fraud and Abuse Act (CFAA) -- United States
- Computer Misuse Act 1990 -- United Kingdom
- General Data Protection Regulation (GDPR) -- European Union
- Convention on Cybercrime (Budapest Convention) -- Council of Europe
- Equivalent cybercrime and data protection laws in your jurisdiction

## Reporting Vulnerabilities

If you discover a security vulnerability in this project or in any system while conducting authorized testing:

1. **Do not exploit the vulnerability** beyond what is necessary to confirm its existence.
2. **Do not publicly disclose** the vulnerability before it has been addressed.
3. **Report the vulnerability responsibly** by contacting the maintainers through a private channel (e.g., a security advisory on the GitHub repository).
4. **Provide sufficient detail** for the issue to be understood and reproduced, including steps, impact, and any suggested remediation.
